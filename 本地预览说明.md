# Hexo本地预览说明

## 基本预览命令

在Hexo中进行本地预览的基本命令是：

```bash
npm run server
```

或者使用简写形式：

```bash
npx hexo server --config _config.yml,_config.local.yml
```

项目里有一个 `_config.local.yml`，会在本地预览时把站点地址覆盖成 `http://localhost:4000`。这样 utterances 登录后会回跳到本地页面，不会跳到线上域名。

如果刚执行过线上构建或部署命令，需要先清掉 Hexo 缓存，否则本地服务可能继续沿用线上 URL。`npm run server` 已经包含这个清理步骤。

## 常用参数

- `-p, --port <port>`: 指定服务器端口，默认为4000
  ```bash
npx hexo server --config _config.yml,_config.local.yml -p 8080  # 使用8080端口
  ```

- `-s, --static`: 仅使用静态文件模式
  ```bash
  npx hexo s --static
  ```

- `-l, --livereload`: 启用自动刷新功能，修改文件后浏览器会自动刷新
  ```bash
npx hexo server --config _config.yml,_config.local.yml --livereload
  ```

- `-o, --open`: 启动服务器后自动打开浏览器
  ```bash
npx hexo server --config _config.yml,_config.local.yml --open
  ```

## 组合使用示例

```bash
# 使用8080端口，并启用自动刷新和自动打开浏览器
npx hexo server --config _config.yml,_config.local.yml -p 8080 --livereload --open
```

## 预览后访问

启动服务器后，通常可以通过以下地址访问：
- http://localhost:4000 (默认端口)
- 或者您指定的端口，如 http://localhost:8080

## 预览注意事项

1. 本地预览不需要部署，修改文件后会自动更新（启用livereload时）
2. 如果修改了配置文件(_config.yml)，需要重启服务器才能生效
3. 完成预览后可以使用 Ctrl+C 停止服务器

## 完整工作流示例

```bash
# 1. 清理缓存
npx hexo clean

# 2. 生成静态文件
npx hexo generate

# 3. 启动本地预览服务器
npx hexo server --config _config.yml,_config.local.yml --livereload --open
```
